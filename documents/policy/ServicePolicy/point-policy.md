## 포인트

### 포인트 기본 정책

- 포인트는 BUYER 회원만 사용할 수 있다.
- 포인트는 1원 단위로 사용할 수 있다.
- 주문 시 포인트 사용 금액은 0원 이상 입력 가능하다.
- 사용 포인트는 사용자의 사용 가능 포인트 잔액을 초과할 수 없다.
- 포인트는 현금으로 환불할 수 없으며, 서비스 내 주문 결제에만 사용할 수 있다.
- 최종 결제 금액 계산식은 다음과 같다.

```
최종 결제 금액 = 상품 금액 - 쿠폰 할인 금액 - 사용 포인트 - 구독 할인 금액 + 배송비
```

- Point balance는 사용자의 현재 사용 가능 포인트 총액을 빠르게 조회하기 위한 요약 값이다.
- PointHistory의 remainingAmount는 FEFO + FIFO 차감 및 만료 처리를 위한 원본 포인트 잔여 금액이다.
- Point balance는 PointHistory의 사용 가능 remainingAmount 합계와 일치해야 한다.
- 포인트 정합성을 검증하는 관리자 메뉴를 사용할 수 있다.
- 포인트 DB의 임의 조작을 방지하기 위해 DB트리거를 설정하고 이를 보안 감사할 수 있는 로그를 활용한다.
- 마이너스 포인트 운용은 적용하지 않는다.

### 포인트 적립 정책

- 포인트는 배송 완료 시 자동 적립된다.
- 적립 금액은 배송비를 제외한 실제 상품 결제 금액의 1%로 계산한다.
- 포인트 사용 금액에 대해서는 포인트를 적립하지 않는다.
- 적립 기준 금액은 다음과 같다.

```
포인트 적립 기준 금액 = 상품 금액 - 쿠폰 할인 금액 - 구독 할인 금액 - 사용 포인트
```

- 적립 포인트는 소수점 이하를 버림 처리한다.

```
적립 포인트 = floor(포인트 적립 기준 금액 × 0.01)
```

- 적립 포인트의 만료일은 적립일 기준 1년 후로 설정한다.
- 적립 포인트는 PointHistory에 EARN 이력으로 기록한다.

### 포인트 적립 시점

- 포인트 적립은 주문의 모든 주문 상품이 배송 완료된 시점에 주문 단위로 1회 처리한다.
- 일부 주문 상품만 배송 완료된 경우에는 포인트를 적립하지 않는다.
- 이미 적립 처리된 주문은 중복 적립하지 않는다.

### 포인트 사용 / 차감 정책

- 주문 생성 또는 결제 승인 시 사용 포인트를 검증한다.
- 사용 포인트는 보유한 사용 가능 포인트보다 클 수 없다.
- 포인트는 만료되지 않았고 잔여 포인트가 남아 있는 적립/충전/복구 이력에서만 차감할 수 있다.
- 포인트 차감은 **FEFO 방식**을 기본으로 한다.
    - FEFO는 First Expired, First Out 방식이다.
    - 즉, 만료일이 가장 가까운 포인트부터 먼저 차감한다.
- 만료일이 같은 포인트가 여러 건인 경우에는 **FIFO 방식**을 보조 정렬로 사용한다.
    - FIFO는 First In, First Out 방식이다.
    - 즉, 같은 만료일이라면 먼저 적립된 포인트부터 먼저 차감한다.
- 포인트 차감 정렬 기준은 다음과 같다.

```
1순위: expireAt ASC
2순위: createdAt ASC

ORDER BY expire_at ASC, created_at ASC
```

- 포인트 차감 시 각 적립/충전/복구 이력의 잔여 포인트를 감소시킨다.
- 포인트 사용 내역은 PointHistory에 USE 이력으로 기록한다.
- USE 이력은 실제 차감된 총 포인트 금액을 음수로 기록한다.
- 포인트 차감은 동시성 충돌 방지를 위해 낙관적 락을 사용한다.
- 낙관적 락 충돌 시 최대 3회까지 재시도한다.
- 3회 재시도 후에도 차감에 실패하면 포인트 차감 실패 예외를 발생시킨다.
- 포인트 차감 실패 시 결제 승인 트랜잭션은 실패하고 롤백된다.
- 현재 Mock 결제 승인 성공 플로우에서는 포인트 차감 실패만으로 주문을 즉시 `CANCELLED`로 자동 전환하지 않는다.
- 포인트 차감 실패 시 Payment는 승인 상태로 저장되지 않는다.
- 결제 실패에 따른 주문 자동 취소, 선점 재고 복구, 쿠폰 복구는 실제 PG 실패 콜백 또는 결제 대기 만료 처리 도입 시 별도 실패 보정 흐름으로 확장한다.

### 포인트 차감 시점

- 주문 생성 시에는 사용 포인트의 입력값과 보유 포인트 초과 여부를 검증한다.
- 결제 승인 시에는 실제 포인트 차감 가능 여부를 재검증한 뒤 차감한다.
- 결제 승인 성공 시 PointHistory에 USE 이력을 기록한다.
- 포인트 차감 실패 시 결제 승인 트랜잭션은 롤백되며, 결제 승인은 완료되지 않는다.
- 현재 Mock 결제 승인 성공 플로우에서는 포인트 차감 실패 시 주문을 즉시 `CANCELLED`로 자동 전환하지 않는다.
- 결제 승인 트랜잭션이 롤백되는 경우 포인트 차감 이력도 함께 롤백되므로 별도 포인트 복구를 수행하지 않는다.
- 실제 PG 실패 콜백 또는 결제 실패 보정 플로우가 도입되어 이미 차감된 포인트가 커밋된 상태에서 결제 실패가 확정되는 경우에만 포인트 복구를 수행한다.
- 동일 주문에 대한 동시 결제 승인 요청이 발생하더라도 Order 비관적 락 기반 중복 결제 방어로 포인트 차감은 1회만 수행된다.

### 포인트 복구 / 환불 정책

- 주문 취소 시 해당 주문에 사용한 포인트는 전액 복구한다.
- 복구된 포인트는 PointHistory에 REFUND 이력으로 기록한다.
- 포인트 복구는 실제 차감되었던 포인트 이력을 기준으로 처리한다.
- 복구 포인트의 만료일은 원래 차감된 포인트의 만료일을 따른다.
- 여러 적립 건에서 포인트가 차감된 경우, 각 차감 건의 만료일을 유지하여 복구한다.
- 단, 원래 만료일이 이미 지난 경우에는 해당 포인트를 사용 가능 포인트로 복구하지 않고 만료 처리한다.
- 포인트 복구는 주문 취소 트랜잭션 안에서 처리한다.
- 포인트 복구 실패 시 주문 취소 전체를 실패 처리한다.

### 포인트 만료 정책 / 관리

- 포인트는 적립일 기준 1년 후 만료된다.
- 포인트 만료 기준 데이터는 DB의 PointHistory에서 관리한다.
- 적립/충전/복구 이력은 expireAt과 remainingAmount를 가진다.
- remainingAmount는 CHARGE / EARN / REFUND 이력에만 사용한다.
- USE / EXPIRE 이력의 remainingAmount는 사용하지 않거나 0으로 저장한다.
- 만료 대상은 expireAt <= 현재 날짜이고 remainingAmount > 0인 포인트 이력이다.
- 만료 처리는 스케줄러 또는 배치가 수행한다.
- 만료 스케줄러는 매월 1일 실행한다.
- 만료 처리 시 해당 적립/충전/복구 이력의 remainingAmount를 0으로 변경한다.
- 만료된 금액만큼 사용자의 Point balance를 차감한다.
- 만료 내역은 PointHistory에 EXPIRE 이력으로 기록한다.
- 포인트 만료 에러시 관리자가 수동으로 진행할 수 있다.

### 테스트용 포인트 충전 API 정책

- 테스트용 포인트 충전 API는 개발/테스트 편의를 위한 임시 API이다.
- 포인트 충전은 BUYER 권한 사용자만 가능하다.
- 1회 최소 충전 금액은 1,000원이다.
- 1회 최대 충전 금액은 1,000,000원이다.
- 충전 금액은 1원 단위로 입력 가능하다.
- 충전된 포인트의 만료일은 충전일 기준 1년 후로 설정한다.
- 충전 내역은 PointHistory에 CHARGE 이력으로 기록한다.
- 운영 환경에서는 해당 API를 비활성화하거나 ADMIN 전용으로 전환한다.

### 포인트 이력 정책

- 모든 포인트 변동은 PointHistory에 기록한다.
- 포인트 이력 유형은 다음과 같다.CHARGE : 테스트용 충전
  EARN : 배송 완료 후 적립
  USE : 주문 결제 시 사용
  REFUND : 주문 취소로 복구
  EXPIRE : 만료
- 포인트 이력에는 다음 정보를 기록한다.
    - 사용자 ID
    - 주문 ID
    - 변동 금액
    - 이력 유형
    - 설명
    - 만료일
    - 생성일
- 사용 또는 만료 이력의 금액은 음수로 저장하고, 적립/충전/복구 이력의 금액은 양수로 저장한다.

### 포인트 조회 정책

- 사용자는 자신의 포인트 잔액과 포인트 이력을 조회할 수 있다.
- 포인트 이력은 최신순으로 조회한다.
- 포인트 이력은 type 조건으로 필터링할 수 있다.
    - CHARGE
    - EARN
    - USE
    - REFUND
    - EXPIRE
- 곧 만료될 포인트는 가장 가까운 만료일 기준으로 합산하여 응답한다.
- 곧 만료될 포인트 기준은 현재일로부터 30일 이내 만료 예정 포인트로 한다.

### 포인트 차감 상세 추적

- 포인트 차감은 FEFO + FIFO 기준으로 수행한다.
- 포인트 사용 시 PointHistory에 USE 이력을 생성한다.
- USE 이력은 사용한 총 포인트 금액을 음수로 기록한다.
- 실제 어떤 적립/충전/복구 이력에서 얼마를 차감했는지는 PointUsageDetail에 기록한다.
- PointUsageDetail은 다음 정보를 가진다.
    - 사용 이력 ID
    - 차감 대상 원본 포인트 이력 ID
    - 차감 금액
    - 원본 포인트 만료일
- 주문 취소 시 PointUsageDetail을 기준으로 실제 차감되었던 포인트를 복구한다.
- 복구 포인트는 원래 차감된 포인트의 만료일을 따른다.
- 원래 만료일이 이미 지난 경우 해당 금액은 사용 가능 포인트로 복구하지 않고 만료 처리한다.
- PointUsageDetail의 원본 포인트 만료일은 주문 취소 시 원래 만료일 기준으로 복구 여부를 판단하기 위해 저장한다.
