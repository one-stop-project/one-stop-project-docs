## 구독

- **구독 상태:**
    - ACTIVE
        - 정상 구독 상태
        - 모든 구독 혜택 사용 가능
        - 자동 결제 대상
        - 재가입 불가
    - CANCELLED (구독 해지 예약에 가까움)
        - 사용자가 구독 해지한 상태
        - 다음 자동 결제 진행하지 않음
        - 환불 없음
        - endAt까지 모든 혜택 사용 가능
        - 재가입 불가
    - EXPIRED
        - 구독 만료 상태
        - 구독 혜택 사용 불가
        - 재가입 가능

- **구독 신청:**
    - 현재 유효한 구독(ACTIVE, CANCELLED)이 존재하는 경우 재가입 불가
    - 구독 신청 시 상태는 ACTIVE로 생성
    - 구독 시작일(startAt)은 가입 시점
    - 구독 만료일(endAt)은 가입 시점 기준 30일 후
    - 다음 자동 결제일(nextPaymentDate)은 최초 endAt과 동일하게 설정
- **구독 해지:**
    - 즉시 해지 처리
    - 해지 즉시 상태를 CANCELLED로 변경
    - 다음 자동 결제 중단
    - 환불 없음
    - endAt까지 혜택 유지
    - 환불 없음
    - 구독 해지 시 사용자가 해지 사유 입력
- **구독 혜택:**
    - 무료 배송
    - 추가 5% 할인
- **자동 결제**
    - 자동 결제는 날짜 기준으로 관리한다.
    - 자동 결제 스케줄러는 매일 자정(00:00)에 실행된다.
    - status가 ACTIVE이고 nextPaymentDate가 오늘 이하인 구독을 자동 결제 대상으로 조회한다.
    - ACTIVE 상태 구독만 자동 결제 대상이다.
    - CANCELLED, EXPIRED 상태 구독은 자동 결제를 수행하지 않는다.
    - 최초 자동 결제일은 가입 시점으로부터 30일 후
    - 자동 결제 성공 시(renew):
        - endAt을 결제 처리 시각(now) 기준 30일 후로 갱신
        - nextPaymentDate를 결제 처리 시각(now) 기준 30일 후로 갱신
        - (기존 endAt/nextPaymentDate 값에 누적 연장하는 방식이 아님 — now + 30일로 재설정)
        - 상태 유지(ACTIVE)
    - 자동 결제 실패 시:
        - 구독 상태를 EXPIRED로 변경
        - 구독 혜택 종료
- **만료 처리:**
    - 구독은 날짜 단위로 관리하므로 시간은 고려하지 않는다.
    - 자동 결제 실패: 즉시 EXPIRED 처리
    - 해지된 구독 만료:
        - 매일 자정 스케줄러 실행
        - endAt이 지난 CANCELLED 구독을 EXPIRED 처리
- **구독 상태 전이**

  ACTIVE → CANCELLED → EXPIRED
  ACTIVE → (자동결제 실패) → EXPIRED
